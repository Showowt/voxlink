# CHECKPOINTS — Five-Push Protocol

This project ships in 5 pushes or fewer. Each push is a checkpoint.

## Push 1/5 — FOUNDATION
- [ ] Project skeleton created
- [ ] Dependencies installed
- [ ] `npm run build` passes locally
- [ ] Deployed to Vercel preview successfully
- [ ] Preview URL loads without errors

## Push 2/5 — CORE
- [ ] Main feature works end-to-end on preview URL
- [ ] Database schema in place (if applicable)
- [ ] Auth flow works (if applicable)
- [ ] Critical user path tested manually

## Push 3/5 — POLISH
- [ ] Design system applied (frontend-design + ui-ux-pro-max)
- [ ] Mobile responsive
- [ ] Edge cases handled (empty states, errors)
- [ ] Loading states for all async operations
- [ ] Forms have validation

## Push 4/5 — HARDEN
- [ ] All env vars set in Vercel dashboard (not via CLI)
- [ ] Error tracking installed (Sentry / LogRocket / etc.)
- [ ] Analytics installed
- [ ] Lighthouse score > 90 on mobile
- [ ] Database has indexes on queried columns

## Push 5/5 — LAUNCH
- [ ] Custom domain configured
- [ ] DNS propagated
- [ ] Final QA pass on production URL
- [ ] Analytics receiving events
- [ ] Go-live announcement drafted
